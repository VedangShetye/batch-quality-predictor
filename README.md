# Batch Quality Risk Predictor

A prototype AI/ML pipeline that predicts whether a manufacturing batch is likely
to pass or fail quality control, based on the process conditions it was made
under — built as a demonstration project for a pharma manufacturing AI/ML role.

Built on **synthetic data** (no real plant data used), designed to reflect
realistic manufacturing relationships so the model has genuine patterns to learn.

---

## What this does

1. **Trains a model** on 1,200 past (simulated) batches to learn which process
   conditions — mixing time, temperature, humidity, moisture, machine, shift —
   predict a QC failure.
2. **Scores new, unseen batches** using that trained model, flagging high-risk
   ones for manual review.
3. **Automatically generates a PDF report and an HTML dashboard** summarizing
   the results — no manual write-up needed.

## Folder structure
```
batch-quality-predictor/
├── requirements.txt
├── README.md
├── scripts/
│   ├── generate_data.py          Day 1: creates training data
│   ├── eda.py                    Day 1: exploratory charts
│   ├── train_model.py            Day 1: trains & saves the model
│   ├── generate_new_batches.py   Day 2: simulates new incoming batches
│   ├── score_batches.py          Day 2: scores new batches with the saved model
│   ├── generate_pdf_report.py    Day 2: builds the PDF report
│   └── build_dashboard.py        Day 2: builds the HTML dashboard
├── data/      <- generated automatically (gitignored)
├── models/    <- generated automatically (gitignored)
├── charts/    <- generated automatically (tracked, so charts render on GitHub)
└── reports/   <- generated automatically (tracked, final PDF + dashboard)
```
`data/` and `models/` are excluded from version control (see `.gitignore`) since
they're fully regenerated by running the scripts — only the code that produces
them needs to be tracked.

## Running the pipeline

### Day 1 — build and train the model
Run in order, from inside `scripts/`:
```
cd scripts
python generate_data.py
python eda.py
python train_model.py
```
This creates `data/batch_data.csv`, exploratory charts in `charts/`, and the
trained model in `models/qc_model.joblib`.

### Day 2 — score new batches and generate reports
Run after Day 1 has completed at least once:
```
python generate_new_batches.py
python score_batches.py
python generate_pdf_report.py
python build_dashboard.py
```
This creates `data/new_batches.csv`, `data/scored_batches.csv`, and the final
deliverables: `reports/batch_qc_report.pdf` and `reports/dashboard.html`.

Open `dashboard.html` in any browser, and the PDF in any PDF viewer — no server
or extra setup needed for either.

---

## Reproducibility

Both `generate_data.py` and `generate_new_batches.py` use fixed random seeds,
so re-running the full pipeline always produces the same results — verified to
match exactly across two different machines during development.

## Model performance (summary)

| Metric | Value |
|---|---|
| Model | Logistic Regression |
| ROC-AUC | 0.706 |
| Recall (Fail class) | 0.627 |
| Decision threshold | 0.553 (tuned, not the default 0.5) |

The decision threshold was deliberately tuned rather than left at the default
0.5 cutoff, since the training data has an imbalanced ~20% failure rate and
catching a true failure matters more than avoiding a false alarm in a QC
context.
